assistant: ### Population/EDU Projection Template for Small WA Communities (<1000)

This template is for small WA communities (<1000 population) projecting population and Equivalent Dwelling Units (EDUs) for sewage works design, per WAC 173-240-060 engineering report requirements [2 p.48; 8 p.49]. Base on 20-year design period [8 p.49]. Use per capita flows of 225 L/capita/d (60 gpd/capita) where household size exceeds national median of 2.6 persons [3 p.99]. Map to engineering report sections (e.g., WAC 173-240-060 (b), (c)) and derive design flows from Table G2-2 [1 p.96]. All projections must use credible data from Ecology-accredited labs and calibrated meters [8 p.49]. Cite historical trends and local land use consistency [2 p.48].

#### Step 1: Base Population Data (OFM Sources)
- **Source**: Use Office of Financial Management (OFM) data for service area population over last two decades (if available) [2 p.48]. Supplement with local land use planning agency data for consistency [2 p.48].
- **Projection Method**: Extrapolate from historical records in approved general sewer plan [2 p.48]. For small communities, assume linear growth from residential population, adding partial EDUs for employment/students (increase linearly at same annual rate as residential) [7 p.15].
- **Template Table** (e.g., for 20-year period, reconcile billing/meter data vs. census for accuracy [unknown direct reconciliation method; use flow monitoring to validate]):

  | Base (e.g., 2023) | X (from OFM) | Adjust for metered flow discrepancies vs. census [8 p.49] | X |
  | +10 yrs | - | - | X + linear growth |
  | +20 yrs | - | - | X + linear growth |
- **Map to Report**: Section (b) - Project description with population data/projections [2 p.48]; Section (c) - Future waste load estimates (20 years) [8 p.49].
- **Uncertainty Bands**: Develop 2+ scenarios (e.g., low/high growth) for uncertainty; use vacancy-adjusted forecasts (0% vacancy overestimates; include rate for accuracy) [11 p.75; 18 p.55].

#### Step 2: Adjustments for Seasonal Population, Vacancy, Institutional Loads
- **Seasonal Population**: Estimate seasonal variations (e.g., transients in motels/campgrounds); convert to full-time equivalents using multipliers or include as institutions [3 p.99; 18 p.55]. Consider flow equalization for peaks [3 p.99].
- **Vacancy**: Adjust forecasts by housing vacancy rate (e.g., exclude for larger forecasts, include for accuracy) [18 p.55].
- **Institutional Loads**: Add partial population equivalents for schools/employment (e.g., schools with showers/cafeteria: 16 gpd/person [1 p.96]; boarding schools: 75 gpd/person [1 p.96]). Include transients/group quarters (e.g., motels: 130 gpd/room at 65 gal/person [1 p.96]).
- **Template Adjustment**:
  - Adjusted Pop. = Base Pop. × (1 - Vacancy Rate) + Seasonal Equiv. + Institutional Equiv.
- **Map to Report**: Section (c) - Wastewater sources distribution (e.g., domestic/commercial/institutional percentages) [2 p.48]; include seasonal/diurnal variations [2 p.48].

#### Step 3: EDU Factors and Conversion
- **EDU Definition**: Convert population to EDUs using per-person factors from Table G2-2 (e.g., dwellings: 100 gpd/person = 1 EDU; trailer courts: 300 gpd/trailer at 3 persons/trailer [1 p.96]).
- **Factors**:
  - Residential: 100 gpd/person or 60 gpd/capita (if >2.6 persons/household) [1 p.96; 3 p.99].
  - Institutional/Commercial: Schools 10-16 gpd/person; motels 130 gpd/room; restaurants 50 gpd/seat [1 p.96].
  - Total EDUs = (Residential Pop. / Persons per DU) + Institutional/Commercial Equiv.
- **Uncertainty Bands**: Apply low/high scenarios (e.g., ±10-20% based on multiple planning forecasts) [11 p.75].
- **Map to Report**: Section (c) - Future waste load/sources, accounting for zoning, industrial expansion, water conservation (increases load concentration) [8 p.49].

#### Step 4: Reconciliation of Data Sources
- Compare billing records (e.g., metered water use) vs. census vs. flow meters; use one full year of data (recommend >1 year) to justify parameters [2 p.48]. Flag discrepancies (e.g., flow >120 gpd/capita indicates I/I [8 p.49]). Reconcile via flow monitoring/location of sampling [8 p.49].
- **Map to Report**: Section (c) - Current conditions analysis with flow/loading data [2 p.48].

#### Step 5: Design Flow Derivations
- **Average Flow (Qavg)**: Population × Unit Flow (e.g., 100 gpd/person [1 p.96]) + I/I (not overlooked in small systems; repair sources [3 p.99]).
- **Peak Flow**: Apply peaking factors; consider wet weather, recirculation [1 p.96]. For seasonal, use modular designs [3 p.99].
- **Map to Report**: Section (g) - Basic design data/sizing [unknown p.; inferred from 45 p.7 context on engineering report]; derive from population projections in (b)/(c) [2 p.48; 8 p.49].
