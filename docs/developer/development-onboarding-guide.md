# Caliper v2 Development Onboarding Guide

This guide provides developers and AI assistants with a fast, accurate map of the repository to enable quick setup, dependency installation, command execution, and immediate contribution.

## Quick Facts
- Python: 3.11 preferred (3.12/3.13 generally OK)
- Package manager: Poetry (installed via pipx)
- Entry script: `caliper_v2` CLI (`tool.poetry.scripts`), implemented in `src/caliper_v2/cli_main.py` and re-exported by `src/caliper_v2/cli/__init__.py`
- Data dir: `data_v2/` (indexes, sqlite, outputs)
- Env file: `.env` (ignored) with template `.env.example`
- Large files: `knowledge_base/` ignored to avoid 100MB GitHub limits
- Line endings: LF for code, CRLF for Windows scripts (via `.gitattributes`)

## Install & Run
1) Ensure Poetry is available (pipx recommended):
   - Windows PowerShell:
     - `py -3.11 -m pip install --user pipx`
     - `py -3.11 -m pipx ensurepath` (restart shell)
     - `pipx install poetry`
   - macOS/Linux:
     - `python3.11 -m pip install --user pipx`
     - `pipx ensurepath && exec "$SHELL" -l`
     - `pipx install poetry`
2) Project setup:
   - `poetry env use 3.11`
   - `poetry install --with llamaindex`
3) Configure environment:
   - `cp .env.example .env` and fill keys (OpenAI, Anthropic, Gemini/Google, xAI, Llama Cloud). `.env` is git-ignored.
4) Smoke test:
   - `poetry run caliper_v2 --help`
   - or `poetry run python run_caliper_v2.py --help`

## Key Files To Review (in priority order)
- `pyproject.toml`: Poetry config, dependencies, script entry point (`caliper_v2 = "caliper_v2.cli:app"`).
- `poetry.lock`: Locked dep versions (generated by Poetry 2.1.4).
- `src/caliper_v2/cli_main.py`: The full Typer CLI implementation (callback, doctor/info, ingest/retrieve flows, retrieval_audit, persistence details, rerankers, hybrid retrieval, summary-layer index, etc.).
- `src/caliper_v2/cli/__init__.py`: Re-exports `app` from `cli_main.py` to satisfy the entry point.
- `src/caliper_v2/core/config.py`: Pydantic `Settings` with paths, provider keys, flags; reads `.env` via pydantic-settings.
- `src/caliper_v2/core/llm_providers.py`: Configures LlamaIndex LLMs (OpenAI, Anthropic, Gemini API/Vertex, xAI Grok) and embeddings.
- `src/caliper_v2/services/persistence.py`: Paths layout, sqlite-backed hash cache for idempotent ingest, v2-specific data/db separation.
- `src/caliper_v2/services/llama_cloud_index.py`: Thin wrapper around the official Llama Cloud SDK (requires `llama-cloud-services`).
- `run_caliper_v2.py`: Simple wrapper that sets `sys.path` and invokes the Typer app.
- `.env.example`: Safe env template to copy to `.env`.
- `.gitattributes`: Line-ending normalization (LF for code, CRLF for Windows scripts), binary/LFS rules, and `archive/**` as `-text`.
- `.gitignore`: Ignores `.env`, venvs, caches, logs, `knowledge_base/`, outputs, etc.

## Environment & Secrets
- The CLI loads `.env` early via `python-dotenv`, then pydantic settings apply defaults (`core/config.py`).
- Critical keys:
  - `OPENAI_API_KEY` (OpenAI models, default embeddings)
  - `ANTHROPIC_API_KEY`
  - `GEMINI_API_KEY` or `GOOGLE_API_KEY` (API mode) OR `GOOGLE_APPLICATION_CREDENTIALS` (Vertex SA mode)
  - `LLAMA_CLOUD_API_KEY` (for Llama Cloud SDK usage)
  - Optional: `XAI_API_KEY`, `COHERE_API_KEY`
- Provider resolution order (in CLI): CLI flags → settings → explicit env override (`CALIPER_LLM_PROVIDER`, `CALIPER_LLM_MODEL`) → auto-detect from env.
- Do not commit real secrets; `.env` is ignored.

## CLI Overview (high level)
- Root callback sets logging, environment, and LLM provider.
- Selected commands present in `cli_main.py` (scan the file):
  - `doctor`: checks API keys, prints provider resolution, lists available indexes under `data_v2/indexes`.
  - `info`: prints a compact runtime settings summary.
  - `retrieval_audit`: takes a retrieval JSON and produces a critique and improvement plan (markdown or JSON).
  - Ingest/Retrieve flows: present within `cli_main.py` (hybrid retrieval, BM25 JSONL rebuild, summary-layer index for HRAG-style routing, optional rerankers).

Suggested discovery:
- Grep for `@app.command()` and Typer options to enumerate all subcommands and arguments.

## Data & Persistence
- Working directory: `data_v2/`
  - Indexes: `data_v2/indexes/<index_name>/` (vector store, summary index, BM25 corpus JSON/JSONL)
  - SQLite db: `data_v2/app_v2.db` (hash cache)
  - Outputs: `outputs/`
- Index pathing is centralized in `services/persistence.py`.
- BM25 is persisted as JSON/JSONL (no pickles required to rebuild deterministically).

## Dependency Strategy
- Poetry manages the environment. Install with `poetry install --with llamaindex`.
- Minimal base deps in `[tool.poetry.dependencies]` plus the `llamaindex` group for LlamaIndex adapters (OpenAI/Anthropic/Gemini, embeddings, readers, vector stores, etc.).
- Python constraint: `>=3.11,<3.14`.

## Git & Repo Hygiene
- Remote may be `caliper_2` (user pivoted); confirm with `git remote -v`.
- Large artifacts: `knowledge_base/` is git-ignored for code-only repos. Use LFS or external storage for PDFs if needed later.
- Line endings normalized via `.gitattributes` to eliminate CRLF/LF churn; `archive/**` marked `-text` to avoid retro-normalization.

## First Moves For Codex
1) Verify environment: `poetry --version`, `python --version`.
2) Install: `poetry env use 3.11 && poetry install --with llamaindex`.
3) Confirm CLI exports: `poetry run caliper_v2 --help`.
4) Run diagnostics: `poetry run caliper_v2 doctor`.
5) If ingest commands are needed, inspect `cli_main.py` for `ingest` and `retrieve` usage, then:
   - Prepare a tiny sample under `knowledge_base/` (untracked by git) or another folder.
   - Ingest with `--persist` to populate `data_v2/indexes/<name>`.
6) If Gemini/Vertex needed, ensure appropriate keys or SA file and install any missing adapter packages the CLI hints at.

## Open Questions / TODOs (handy for Codex)
- Confirm which LLM provider the user prefers for default runs; wire defaults accordingly via `.env`.
- Validate that `llama-cloud-services` is available when cloud index features are used; otherwise gate commands with clearer errors.
- Consider adding a `README` section listing all CLI commands (Typer auto-generates help; we can mirror it).
- Consider pre-commit for formatting/lint (repo includes `.pre-commit-config.yaml`).

## Troubleshooting
- Pipx not recognized (Windows): reopen PowerShell or use `py -3.11 -m pipx ...`; ensure `%USERPROFILE%\AppData\Roaming\Python\Python311\Scripts` is on PATH.
- Git push blocked by large files: keep `knowledge_base/` ignored (already in `.gitignore`) or migrate PDFs to LFS and rewrite history.
- CRLF/LF warnings in Desktop: normal; `.gitattributes` addresses normalization. Optionally `git add --renormalize .` once.
